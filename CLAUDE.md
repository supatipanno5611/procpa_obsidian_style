## Project

Personal blog / knowledge base for a Korean CPA (procpa.co.kr). Rebuild of an existing site — the old codebase lives under `reference/` and is **content-only** (copy text, hrefs, factual data may be reused; **design and component code must not be copied**). Content authoring flow: author writes Markdown in Obsidian and drops files into `content/` for the site to pick up. Two content formats coexist:

- **posts** — standalone blog entries (`content/posts/*.md`)
- **series** — ebook-style multi-chapter works (`content/series/<slug>/index.md` for the series meta, `content/series/<slug>/NN-*.md` for chapters)

Both carry a `category: 'accounting' | 'ai'` frontmatter field that drives category-split UI on the home page.

## Commands

```bash
npm run dev         # predev runs velite; then next dev (Turbopack)
npm run build       # prebuild runs velite; then next build
npm run content     # velite once
npm run content:watch  # velite in watch mode (when editing content alongside dev)
npm run lint        # eslint
```

After editing any file in `content/`, Velite must rebuild. `predev`/`prebuild` hooks handle this automatically; if the dev server is already running, either restart it or run `npm run content:watch` in a second terminal.

Vercel CLI (installed globally) is the deploy path: `vercel` for preview, `vercel env pull` for env. **Do not `vercel --prod` yet** — procpa.co.kr is still served by the old site; production cutover will happen as a separate, deliberate step.

## Content pipeline (Velite)

`velite.config.ts` defines three collections: `posts`, `series`, `chapters`. Velite reads `content/`, validates frontmatter with Zod, compiles MDX to a JS function string, and emits typed JSON + a TypeScript declaration to `.velite/`. The output is consumed via the path alias `#site/content` (declared in `tsconfig.json`), e.g.:

```ts
import { posts, series, chapters } from '#site/content'
```

Key points when touching the pipeline:

- Pattern collision matters. `chapters` uses `['series/**/*.md', '!series/**/index.md']` specifically so it does not swallow series meta files. If you rename `index.md` or change patterns, confirm `.velite/series.json` and `.velite/chapters.json` are both non-empty.
- MDX `body` is a **function-string**, not HTML. Render via `src/components/mdx-content.tsx`, which `new Function(code)`s it with `react/jsx-runtime` at render time.
- `post.metadata.readingTime` / `toc` / `tags` are computed by Velite — don't recompute in the UI.
- `slugAsParams` is the post-transform route-friendly slug; `slug` is the raw file path. Use `slugAsParams` for routing, `slug` as a stable id.

## Routing architecture

App Router under `src/app/`. Next.js 16 conventions apply everywhere — **`params` and `searchParams` are `Promise`** and must be `await`ed (the build will error otherwise; see `opengraph-image.tsx` fix history). Middleware is `proxy.ts` (not used yet, but don't reintroduce `middleware.ts`).

Critical routing rules specific to this repo:

- **Post detail is `app/posts/[...slug]/page.tsx`** (catch-all) so that nested post folders work. Next.js forbids file conventions (`opengraph-image.tsx`, etc.) after catch-all segments — dynamic OG for posts and chapters is generated by `src/app/api/og/route.tsx` and linked via `generateMetadata().openGraph.images`, **not** by a per-route `opengraph-image.tsx`. The root `src/app/opengraph-image.tsx` and the series-landing `src/app/series/[series]/opengraph-image.tsx` are fine because they are not under a catch-all.
- **Search** has two implementations that share one data source (`src/lib/search.ts` → `getSearchDocs()`):
  1. `src/app/search-index.json/route.ts` — `force-static` JSON consumed by the client-side `CommandPalette` (`src/components/command-palette.tsx`) which runs MiniSearch in the browser. When passing results to `<CommandItem>`, the `value` prop must include searchable text (title/description/tags), not just the id — cmdk's built-in filter matches `value` as a substring and will drop all results for Korean queries if `value` is an ASCII slug.
  2. `src/app/search/page.tsx` — SSR advanced search with `q`, `type`, `tag`, `from`, `to` URL params.
- `.velite` and `reference/` are in `tsconfig.json`'s `exclude` list. `reference/` contains the old site's code and must stay excluded or the build will fail on its stray imports.

## Design system

- **Framework**: Tailwind v4 + shadcn/ui (zinc base). Plugins are registered via `@plugin` in `src/app/globals.css` (not via tailwind config). Typography plugin is the only one added beyond shadcn defaults.
- **Fonts**: Pretendard (CDN `<link>` in `src/app/layout.tsx` head) for sans, Geist Mono (`next/font/google`) for mono / labels / numbers.
- **Theme**: pure-black dark (`oklch(0 0 0)`) + blue primary. Dark is default. There is **no next-themes** — React 19 warns about scripts rendered inside components, so the pre-hydration theme script lives in `src/lib/theme-script.ts` and is injected directly into `<head>` via `dangerouslySetInnerHTML`. `src/components/theme-toggle.tsx` toggles the `dark` class on `<html>` and writes `localStorage('theme')` manually.
- **Layout width**: header, footer, and home all use `max-w-6xl` — keep them in sync when adjusting widths so the site's left/right edges line up.
- **Tone rule**: no gradients, no glassmorphism, no emojis in UI. Hierarchy comes from typography, monospace labels, and hairline dividers (`border-border/60`). Korean text gets `word-break: keep-all` globally (set on `<body>` in the layout).

## Home page design variants

There are currently two home pages live:

- `src/app/page.tsx` — v2 "editorial" home (Hero → Philosophy → Series → Posts with category split).
- `src/app/home-c/page.tsx` — Vault/Obsidian-style variant (sidebar + note view). This is the preferred direction and will eventually replace `/`. The sidebar content is in `src/components/home-c/vault-sidebar.tsx` and is reused by `src/components/home-c/vault-drawer.tsx` (shadcn `Sheet` triggered on `< lg` screens so the sidebar doesn't dominate mobile first paint).

When promoting `/home-c` to `/`, move the page body into `src/app/page.tsx` and delete `src/app/home-c/`.

## SEO

- `generateMetadata()` in post and chapter pages sets `openGraph.images` / `twitter.images` to `/api/og?kicker=&title=&subtitle=&meta=`.
- JSON-LD (`BlogPosting` + `BreadcrumbList`) is injected via `src/components/json-ld.tsx` — keep the `SITE` constant in each page in sync with `metadata.metadataBase` in `src/app/layout.tsx`.
- `sitemap.ts`, `robots.ts`, `rss.xml/route.ts` all read directly from `#site/content`, so new content is picked up on the next build.
