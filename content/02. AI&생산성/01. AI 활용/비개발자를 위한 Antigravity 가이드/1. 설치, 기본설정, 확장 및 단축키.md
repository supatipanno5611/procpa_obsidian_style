## 1. Antigravity 설치 및 첫 실행
### 1.1. 프로그램 다운로드 및 설치
먼저 [Antigravity 공식 웹사이트](https://antigravity.google/download)를 방문하여 자신의 운영체제(macOS, Windows 등)에 맞는 설치 파일을 다운로드합니다. 

![](https://i.imgur.com/6z2DvSJ.png)

설치 과정은 일반적인 소프트웨어와 동일하므로 생략합니다.

### 1.2. 셋업
설치 완료 후 처음으로 실행하면 setup 화면이 나옵니다. IDE 사용이 처음이라면 default 설정대로 셋업하는 것을 추천합니다.(본 글의 세팅 과정에서 더 자세히 다룰 예정입니다.)

![](https://i.imgur.com/QJWYJ7b.png)

![](https://i.imgur.com/6PGYNmT.png)

![](https://i.imgur.com/QL3vYuT.png)

셋업을 마치면 구글 계정 로그인 창이 나타납니다. 평소 사용하시는 구글 계정으로 로그인만 하면 모든 준비가 끝납니다.

### 1.3. 첫  실행
처음 실행하면 나타나는 화면의 각 창을 살펴 보겠습니다.(버전에 따라 향후 달라질 수 있으나, 기본적으로 아래와 같은 화면이 나타납니다.)

![](https://i.imgur.com/yODLGqx.png)

1. **Open Folder**: 작업을 진행할 폴더를 선택해야 합니다. 이를 **워크스페이스(Workspace)** 라고 부르는데요. Antigravity는 기본적으로 선택된 폴더 내에서만 파일들을 분석하여 최적의 답변을 내리기 때문에, 프로젝트별로 전용 폴더를 만들어 관리하는 것이 좋습니다.
2. **Open Agent Manager**: Antigravity의 핵심인 '에이전트'들을 한눈에 관리하는 곳입니다. 여러 개의 프로젝트(워크스페이스)를 동시에 띄워놓고 에이전트들이 일하는 모습을 지켜보거나, 지난 대화 기록을 찾아볼 때 사용합니다.
3. **Clone Repository**: GitHub[^1] 에 올려둔 내 프로젝트가 있다면, 그 주소를 복사해와서 내 컴퓨터로 가져오는 기능입니다. 

>[!tip]
> 에이전트 모드(Agent mode)를 활용하면 여러 에이전트를 병렬로 실행할 수 있어 생산성이 크게 향상됩니다. 하지만 처음 접하시는 분들께는 폴더(Folder)를 직접 열어 사용하는 방식부터 시작해 보시는 것을 추천드립니다.

## 2. Antigravity 핵심 세팅
Antigravity Settings에는 수많은 항목이 존재하지만, 여기서는 비개발자분들의 눈높이에 맞춰 초기 사용 시 반드시 확인해야 할 **핵심 설정**들만 골라 정리해 보겠습니다.

우선, `Ctrl + ,` 를 눌러서 설정을 엽니다.

### 2.1. Agent
에이전트[^2]가 내 컴퓨터에서 얼마나 자유롭게 활동할 수 있게 할지를 결정하는 설정입니다.

#### 2.1.1. Artifact - Review Policy
에이전트가 파일의 내용을 수정하거나 생성할 때, 사용자가 이를 매번 검토할지 여부를 결정하는 설정입니다.

![](https://i.imgur.com/aOAyOFJ.png)


- **Always Proceed** : 별도의 승인 절차 없이 에이전트가 모든 파일 수정 사항을 즉시 반영합니다. 작업 흐름이 끊기지 않아 속도가 매우 빠르지만, 의도치 않은 코드 변경이 발생할 위험이 있습니다.

- **Agent Decides (추천)** : 에이전트가 스스로 판단하여 중요한 변경 사항이 있을 때만 사용자에게 검토를 요청합니다. 불필요한 클릭을 줄여 생산성을 높여줍니다.

- **Request Review** : 에이전트가 수행하는 모든 파일 수정 작업을 사용자가 직접 확인하고 승인해야 합니다. 작업의 정확도를 극대화할 수 있지만, 번거롭습니다.

#### 2.1.2. Terminal - Terminal Command Auto Execution
가장 주의해야 할 부분입니다. 터미널[^3]에서 명령을 실행할 때 사용자의 승인을 받을지 여부를 결정합니다.

![](https://i.imgur.com/peNRlBa.png)

- **Request Review (추천)** : 에이전트가 어떤 명령을 내리기 전, 반드시 사용자에게 승인을 요청합니다. 처음 사용한다면 '안전 제일'을 추구하는 입장에서 처음에는 이 설정을 가장 추천합니다.

- **Always Proceed** : 승인 절차 없이 에이전트가 모든 명령을 바로 실행합니다. 작업 속도는 빠르지만, 의도치 않은 파일 수정 사고가 발생할 위험이 있습니다.

>[!WARNING]
> 최근의 모델들은 성능이 매우 뛰어나 'Always Proceed' 모드에서도 파일을 전체 삭제하거나 엉뚱하게 수정하는 등의 치명적인 오류는 잘 발생하지 않습니다. 하지만 만약의 상황을 대비해 중요한 작업을 시작하기 전에는 반드시 Git 커밋[^4] 을 해두는 습관을 기르는 것이 좋습니다. Git에 대한 자세한 활용법은 추후 별도 포스팅에서 다루겠습니다.

#### 2.1.3. Automation - Agent Auto-Fix Lints
에이전트가 생성한 코드의 린트 에러(예: 코드 스타일 위반, 잠재적 버그 경고)를 자동으로 수정하는 기능입니다. Python, Javascript 등 주요 언어의 확장(extension)과 연동되어 작동합니다. 활성화 하는 것을 추천합니다.

![](https://i.imgur.com/inonehP.png)

### 2.2. Tab
Tab 기능은 코드를 작성하는 도중 AI가 다음에 올 내용을 미리 예측하여 제안해 주는 **인공지능 코드 완성** 기능입니다. 단순히 단어를 추천하는 수준을 넘어, 현재 작성 중인 코드의 문맥을 파악하여 다음에 올 로직을 통째로 제안하기도 합니다.

작업 속도를 비약적으로 높여주므로 모든 기능을 반드시 **활성화** 상태로 사용하는 것을 추천합니다.

![](https://i.imgur.com/Q2Wx4Ys.png)


## 3. 확장(Extension) 세팅
Antigravity는 그 자체로도 강력하지만, 몇 가지 확장 프로그램(Extension)을 더하면 실무에서의 활용도가 비약적으로 높아집니다. 제가 직접 사용해보고 꼭 필요하다고 느낀 3가지를 정리해 드립니다.

### 3.1. 한국어 언어 팩 (Korean Language Pack)

Antigravity는 기본적으로 영문 메뉴로 구성되어 있습니다. 저처럼 영어에 약하신 분들께는 한국어가 훨씬 편안하실 겁니다. 다행히 훌륭한 개발자분들이 만들어주신 한국어 언어 팩이 있습니다.

![한국어 언어 팩 설치 화면](https://i.imgur.com/Q8224cc.png)

1. **실행**: 좌측 사이드바에서 사각형 모양의 'Extensions' 아이콘을 클릭하거나 단축키 `Cmd + Shift + X`를 누릅니다.

2. **검색 및 설치**: 검색창에 `Korean Language Pack`을 입력하고 'Install' 버튼을 클릭합니다.

3. **재시작**: 설치가 완료되면 우측 하단에 'Change Language and restart'라는 안내 창이 뜹니다. 이를 클릭하여 프로그램을 재시작하면 대부분의 메뉴가 한국어로 변경됩니다.

### 3.2. 사용량 모니터링 (Antigravity Cockpit)

무료 버전이라 하더라도 사용량 제한은 존재합니다. 내가 오늘 얼마나 에이전트를 부려먹었는지 궁금할 때, 실시간으로 잔여량을 확인할 수 있는 도구가 바로 `Antigravity Cockpit`입니다.

![Antigravity Cockpit 사용량 모니터 화면](https://i.imgur.com/jNbiehP.png)

1. **설치**: Extensions 탭에서 `Antigravity Cockpit`을 검색하여 설치합니다.

2. **확인**: 설치 후 화면 하단 상태바를 보시면 사용량이 백분율(%)로 표시됩니다.

3. **상세 설정**: 이 수치를 클릭하면 상세 모니터 화면이 나타납니다. 여기서 모든 모델의 체크박스를 활성화해두면 모델별 잔여량을 한눈에 관리할 수 있어 업무 조절에 큰 도움이 됩니다.

### 3.3. 마크다운(Markdown) 창작 도구

저처럼 옵시디언(Obsidian) 등 마크다운 편집기를 활용해 글을 작성하는 분들께 추천드리는 확장 프로그램입니다.

#### 3.3.1. Markdown All in One

옵시디언이나 일반 마크다운 에디터에서 쓰던 편리한 단축키들을 그대로 가져올 수 있습니다.

- **주요 기능**: `Cmd + B`로 굵게 표시하기, 엔터 시 자동으로 다음 리스트 생성 등 문서 작성을 훨씬 쾌적하게 만들어줍니다.

![Markdown All in One 설정 화면](https://i.imgur.com/bznBh5T.png)

#### 3.3.2. Markdown Preview

작성 중인 문서가 실제로 어떻게 보일지 실시간으로 미리보기 기능을 제공합니다.

- **주요 기능**: 편집기에서 텍스트만 보면 가독성이 떨어지는데, 이 기능을 켜두면 실제 발행될 결과물을 옆에 띄워두고 작업할 수 있습니다.

[[TIP]]
옵시디언과 파일을 동시에 작업하다 보면 간혹 동기화 과정에서 충돌이 생길 수 있는데, 가급적 Antigravity 내에서 수정을 끝내고 프리뷰로 확인하는 습관을 들이시는 것이 안전합니다.
[[/TIP]]

![Markdown Preview 실행 화면](https://i.imgur.com/n3Bqd7A.png)

## 4. 필수 단축키 (shortcut)

업무 속도를 높여줄 핵심 단축키를 정리해 보았습니다.

- **화면 전환**: `Cmd + E` (Win: `Ctrl + E`) - 매니저 모드와 에디터 모드를 오가는 핵심키

- **AI 지시**: `Cmd + L` (Win: `Ctrl + L`) - 현재 위치에서 AI에게 바로 말을 거는 키

- **AI 편집**: `Cmd + I` (Win: `Ctrl + I`) - 편집기 내에서 선택한 코드에 대해 즉시 수정을 요청하는 키

- **파일 찾기**: `Cmd + P` (Win: `Ctrl + P`) - 파일 이름으로 프로젝트 내 파일을 빠르게 찾아 여는 키

> [!Tip]
> 기본 단축키 외에도 사용자의 편의에 맞춰 모든 기능을 커스텀 단축키로 지정할 수 있습니다.
> 
> 1. **설정 방법**: `Cmd + K`를 누른 후 연달아 `Cmd + S`를 누르면 전체 단축키 목록 창이 열립니다.
> 2. **검색 및 수정**: 상단 검색창에 변경하고 싶은 기능의 이름(예: 'Side Bar')을 입력하고, 해당 항목을 더블 클릭하여 원하는 키 조합을 입력하면 즉시 반영됩니다.
> 3. **활용 예시**: 저의 경우 왼쪽 사이드바를 여닫는 동작을 옵시디언과 동일하게 `Shift + Alt + ←`로 설정하여 사용 중입니다.


[^1]: **GitHub**: 전 세계 개발자들이 코드를 저장하고 공유하는 온라인 저장소입니다. '코드 전용 구글 드라이브' 혹은 '개발자들의 도서관'이라고 이해하면 쉽습니다.
[^2]: **에이전트(Agent)**: 내 명령을 이해하고 실제로 파일을 수정하거나 터미널을 조작하는 AI 직원입니다.
[^3]: **터미널(Terminal)**: 컴퓨터 시스템에 직접 문자로 명령을 내리는 창입니다. 에이전트의 '손'과 같은 역할을 합니다.
[^4]: **커밋(Commit)**: 작업 내용을 GitHub에 영구적으로 기록하는 저장 행위입니다. 만약 에이전트가 실수하더라도 이전 커밋 시점으로 되돌릴 수 있는 '저장 포인트' 역할을 합니다.
