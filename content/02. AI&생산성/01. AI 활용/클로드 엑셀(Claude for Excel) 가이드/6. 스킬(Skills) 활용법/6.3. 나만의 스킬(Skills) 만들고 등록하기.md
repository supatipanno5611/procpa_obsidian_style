스타터 스킬도 유용하지만, 스킬의 진정한 위력은 **내 업무에 필요한 전문 지식과 작업 프로세스, 그리고 데이터 양식을 직접 만들어 추가할 수 있다**는 점에 있습니다. 범용 AI 에이전트가 알 수 없는 실무 로직이나 산출물 양식 등을 스킬로 정의해 두면, 오류를 줄이고 일관성 있는 결과를 보장하여 훨씬 다양한 업무 영역으로 확장할 수 있습니다.

## 1. 스킬(Skill)을 만드는 스킬 : skill-creator

스킬을 만들기 위해서 복잡한 코딩을 배울 필요는 없습니다. 클로드에는 스킬을 만들어주는 전용 에이전트인 `skill-creator`가 내장되어 있어, 단순히 대화하는 것만으로 스킬을 만들 수 있습니다.

### 1.1. 아이디어로 스킬 만들기

아무것도 없는 백지상태에서 내가 구상한 아이디어를 스킬로 만들어야 할 때 사용하는 정석적인 방법입니다. 
자연어로 "○○ 작업을 하는 스킬을 만들어줘"라고 지시하거나, 채팅창에 `/skill-creator` 슬래시 명령어를 명시적으로 입력하여 시작할 수 있습니다. (두 가지 방식 모두 결과적으로 클로드 내부에 탑재된 `skill-creator` 스킬을 사용하는 구조라 기능적으로는 동일합니다.)

- **자연어 예시:** "계정별 원장을 통합, 가공 후 Pivot 테이블로 정리하는 스킬 만들어줘"
- **명령어 예시:** `/skill-creator 계정별 원장 통합 및 가공 후 Pivot 테이블로 정리`

명령이 떨어지면 스킬 크리에이터가 단계별로 안내해주며, 내가 원하는 기능에 맞는 스킬의 이름, 언제 이 스킬을 호출할 것인지(트리거), 그리고 세부 마크다운 작업 지침서를 체계적으로 설계하고 구성해 줍니다. 

### 1.2. 기존 작업 과정(세션)을 스킬로 변환하기

개인적으로 가장 추천하는 방법으로, 경험상 스킬의 완성도가 가장 높은 방법입니다.
클로드 엑셀과 함께 엎치락뒤치락하며 고생해서 데이터를 분석해내거나 템플릿을 완성했다면, **그 대화창(세션)에서 한 작업을 그대로 스킬로 만들어달라**고 요청하는 것입니다.

- 예시: "방금 채팅창에서 했던 데이터 가공 과정과 피벗테이블 생성 작업을 새로운 스킬로 만들어줘.

이 경우, 시행착오 과정과 그걸 에이전트가 어떻게 해결했는지에 대한 생생한 컨텍스트가 챗봇의 기억에 고스란히 남아있기 때문에, 만들어지는 스킬의 에러 대처 능력이 엄청나게 올라갑니다.


> [!note] 커스텀 스킬(Custom Skills)과 로컬 스킬(Local Skills)
> 
> 클로드 생태계에서 스킬은 저장 방식에 따라 두 가지로 나뉩니다.
> 
> - **커스텀 스킬:** 앞서 `skill-creator`로 만든 방식처럼 내 엔트로픽(Anthropic) 클라우드 계정에 저장되는 스킬입니다. 로그인만 하면 클로드 엑셀, 클로드 데스크톱 앱 등 **어디서든** 불러올 수 있습니다.
> - **로컬 스킬:** 스킬을 내 PC의 폴더에 저장해두고 불러다 쓰는 방식입니다. Antigravity, 클로드코드와 같이 **로컬 환경에서** 작업할 때 사용합니다.
> 
> 현재 **클로드 엑셀에서는 커스텀 스킬만을 지원합니다.** 따라서 엑셀 안에서 나만의 스킬을 자동화하려면 반드시 클라우드에 저장해 사용해야 합니다.

### 1.3. /skill-creator 상세 사용법

`/skill-creator`는 클로드에 기본 내장된 메타 스킬입니다. 스킬을 만들어주기도 하지만 이제는 **기존 스킬의 업그레이드 및 성능 평가(Eval)까지**할 수 있습니다.

#### 1.3.1. 권장 프롬프트 패턴 예시

`/skill-creator` 뒤에 자연어로 원하는 작업을 설명하면 됩니다. 정해진 명령어 문법이 있는 것은 아니지만, 아래 패턴으로 입력하면 의도한 기능이 잘 작동합니다.

| 명령어 패턴                        | 용도                | 사용 예시                                                                   |
| ----------------------------- | ----------------- | ----------------------------------------------------------------------- |
| `/skill-creator [설명]`         | 새 스킬 생성 (초안 설계)   | `/skill-creator 거래처별 월간 청구서를 정리하는 스킬을 만들어줘`                             |
| `/skill-creator [스킬명] 수정해줘`   | 기존 스킬 편잡 및 추가     | `/skill-creator monthly-report 스킬 워크플로우 3단계에 원형 차트 생성 기능도 추가해줘`         |
| `/skill-creator [스킬명] 테스트해줘`  | 가상 성능 평가(Eval) 실행 | `/skill-creator 이 스킬이 다양한 양식에도 일관되게 적용되는지 성능을 테스트해줘`                    |
| `/skill-creator [스킬명] 설명 최적화` | 호출(트리거) 상황 정확도 개선 | `/skill-creator 사용자가 대충 '월말 결산'이라고만 쳐도 작동하게 스킬의 Description을 똑똑하게 다듬어줘` |

#### 1.3.2. 스킬 성능 향상을 위한 텍스트 구성 팁

스킬 크리에이터가 채팅으로 물어보는 항목에 대해 답변할 때 아래의 팁을 참고하여 적어주시면 훨씬 완벽한 스킬이 구축됩니다.

| 스킬 핵심 구성 요소              | 작성 팁 (Tip)                                                                                                                                   |
| ------------------------ | -------------------------------------------------------------------------------------------------------------------------------------------- |
| **명확한 이름 (Name)**        | 스킬의 용도를 영문(대시 하이픈 연결)으로 명확히 적으세요. (예: `weekly-excel-report`, `sales-dashboard`)                                                              |
| **상세한 설명 (Description)** | 엑셀에서 사용자가 채팅을 쳤을 때 **어떤 키워드와 상황에서 이 스킬이 자동으로 튀어나올지(트리거)** 결정짓는 매우 중요한 문장입니다. 타겟을 아주 구체적으로 적어주세요.                                             |
| **순차적인 워크플로우 지침**        | 에이전트가 엑셀을 조작할 단계적 순서입니다. 모호하게 뭉뚱그리지 말고 `1단계 (원시 컬럼명 검증)`, `2단계 (특정 조건 필터링)`, `3단계 (피벗테이블 추출)`, `4단계 (조건부 서식 적용)` 처럼 잘게 분할할수록 스킬의 완성도가 높아집니다. |
| **입출력 포맷(Format) 정의**    | 입력받을 외부 원본 데이터 파일의 생김새(헤더명 등)와 결과물의 엑셀 보고서 양식을 분명하게 명시하세요.                                                                                   |
| **예외 상황과 에러 처리**         | "만약 11월 매출액 데이터가 텅 비어있다면 에러를 뱉지 말고 해당 셀을 노란색으로 칠해버려" 등 구체적으로 예외 처리를 지시하면 스킬의 안정성이 보장됩니다.                                                     |

## 2. 커스텀 스킬로 등록하기

클로드 엑셀 대화창에서 스킬을 성공적으로 만들어냈더라도, 이 상태로는 해당 세션(대화창) 안에서만 사용할 수 있습니다. 아쉽게도 현재의 클로드 엑셀 Preview 버전에서는 완료된 스킬을 내 클라우드 계정에 자동 저장해 주는 기능이 아직 존재하지 않습니다.

따라서 내가 새로 만든 스킬을 계속해서 불러와 사용하려면 클로드 웹사이트(Claude.ai)나 클로드 데스크톱 앱에 아래의 방법으로 직접 등록해 주어야 합니다.

**1. 완성된 스킬 텍스트 출력 요청**

클로드 엑셀 채팅창에서 **"방금 만든 스킬의 제목, 설명, 지침 전문을 채팅창에 그대로 출력해 달라"** 고 요청합니다. 
![스킬 텍스트 복사 화면](https://raw.githubusercontent.com/procpalee/my_obsidian_images/master/20260324222756682.png)

**2. 클로드 웹/앱에서 스킬 지침 작성으로 이동**

브라우저에서 `Claude.ai`에 접속하거나 클로드 데스크톱 앱의 프로필 메뉴에서 `사용자 지정` ➔ `스킬` ➔ 우측 상단의 `+` ➔ `스킬 지침 작성` 버튼을 차례로 클릭합니다.
![스킬 추가 화면 진입](https://raw.githubusercontent.com/procpalee/my_obsidian_images/master/20260324221904446.png)

**3. 스킬 텍스트 붙여넣기 및 만들기**

스킬 작성 화면이 열리면, 앞서 엑셀 채팅창에서 복사해 둔 스킬 텍스트를 그대로 붙여넣기 한 뒤, 우측 하단의 `만들기` 버튼을 클릭합니다.
![스킬 붙여넣기 및 만들기](https://raw.githubusercontent.com/procpalee/my_obsidian_images/master/20260324223237011.png)

**4. 스킬 생성 완료**

클라우드 계정에 나만의 커스텀 스킬이 정식으로 생성 및 보관됩니다.
![스킬 생성 완료 화면](https://raw.githubusercontent.com/procpalee/my_obsidian_images/master/20260324223403477.png)

**5. 클로드 엑셀에서 즉시 연동 확인**

이제 엑셀로 다시 돌아와서 확인해보면 방금 등록한 나의 커스텀 스킬이 클로드 엑셀에도 등록되어 있음을 확인할 수 있습니다.
![클로드 엑셀 연동 확인 화면](https://raw.githubusercontent.com/procpalee/my_obsidian_images/master/20260324223547253.png)

> [!info] 향후 업데이트 기대
> 
> 나만의 스킬을 번거롭게 웹으로 직접 옮겨 심어야 하는 수동 등록 과정은 Preview 버전의 일시적인 기능 제약입니다. 향후 정식 버전에서는 클로드 엑셀 인터페이스 안에서 생성된 스킬을 즉시 자동 등록하는 기능이 업데이트될 것으로 예상됩니다.
