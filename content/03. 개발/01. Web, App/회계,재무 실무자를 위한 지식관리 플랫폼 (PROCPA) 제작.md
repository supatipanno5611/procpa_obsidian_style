최근 클로드코드(Claude Code)와 안티그래비티(Antigravity)를 활용하여 지식 관리 웹사이트를 새롭게 구축했습니다. 이 글에서는 그 제작 과정과 사용 기술, 주요 기능들을 공유하고자 합니다. 저처럼 개인 블로그나 지식 관리용 웹사이트를 직접 만들어보고 싶은 비개발자 분들에게 작은 도움이 되기를 바랍니다.

## 제작 동기
기존에 워드프레스 블로그를 운영하면서 몇 가지 아쉬운 점이 있었습니다.

- **마크다운 활용 한계**: 오랜 기간 옵시디언(Obsidian)을 통해 개인 지식을 마크다운(Markdown) 기반으로 관리해 왔습니다. 워드프레스에 본문을 복사해서 붙여넣을 수는 있었지만, 옵시디언의 유용한 메타데이터를 제대로 활용할 수 없다는 한계가 있었습니다.
- **개발 환경의 제약**: 워드프레스는 PHP 기반이며 백엔드 서버를 직접 통제하기 어렵습니다. 프론트엔드 디자인을 로컬 환경에서 즉각적으로 테스트하고 수정하는 것도 상당히 번거로웠습니다.
- **제한적인 확장성과 커스터마이징**: 웹사이트에 다양한 커스텀 기능을 추가하고 싶었으나, 플러그인을 일일이 찾아 설치하고 사용법을 익히는 방식에는 한계가 있었습니다. 주어진 템플릿의 제약에서 벗어나 제 입맛에 맞게 직접 개발하고 싶었습니다.
- **AI 코딩 어시스턴트 활용 욕구**: 무엇보다 안티그래비티나 클로드코드와 같은 강력한 AI 에이전트를 실제로 활용해, 완성도 높은 나만의 사이트를 직접 운영해보고 싶은 마음이 가장 컸습니다.

> [!info] 기존 대안들과의 차별점
> 물론 Obsidian Publish, Quartz, Hugo 등 여러 정적 사이트 생성기(SSG) 대안이 존재합니다. 하지만 저는 단순한 개인 블로그를 넘어, 제가 가진 실무 지식을 더 입체적으로 공유할 수 있는 '지식 데이터베이스'를 원했습니다. 이를 위해 유연하게 여러 기능을 덧붙일 수 있어야 했고, 디자인 역시 온전히 제가 통제하여 구현하고 싶었습니다.

## 사용한 도구: 클로드코드 + 안티그래비티
이번 프로젝트는 두 가지 AI 코딩 에이전트를 조합하여 진행했습니다.

### 클로드코드: 기본 구조 설계
현시점 개발 분야에서 독보적인 퍼포먼스를 보여주는 클로드코드에게 기획안을 전달하여 초기 뼈대를 잡았습니다. 원하는 기능과 디자인 요건을 정리한 뒤 '계획(Plan)' 모드를 통해 클로드코드와 대화하며 프로젝트의 구체적인 틀을 완성해 나갔습니다.

### 안티그래비티: 디자인 및 디테일 수정
클로드코드로 기본 구조를 세팅한 후, 안티그래비티를 투입하여 세부 디자인과 디테일들을 수정했습니다. 구글(Google)의 모델들이 시각적인 UI/UX 구현에 강점이 있다는 평가가 많은 만큼, 디자인 작업은 전적으로 안티그래비티(특히 stitch MCP 기능 적극 활용)를 통해 정교하게 다듬었습니다.

## 기술 스택
이 프로젝트는 현재 가장 널리 쓰이는 모던 프론트엔드 기술을 적극 차용하여, 관리의 편의성과 쾌적한 퍼포먼스, 빼어난 디자인을 모두 챙길 수 있도록 구축했습니다.

- Core Framework: Next.js 16 (App Router), React 19, TypeScript
- Styling & UI: Tailwind CSS v4, Radix UI, Framer Motion (애니메이션 탑재용), Lucide React (아이콘)
- Content Data Layer: Velite (마크다운/MDX를 정적 데이터 파이프라인으로 변환)
- Search Engine: Pagefind (별도의 서버 없이 뛰어난 성능을 보장하는 정적 사이트 전역 검색)
- Database / Backend: Supabase (뉴스레터 구독 등 동적 데이터 처리가 필요한 영역에 제한적으로 사용)

여기서 가장 핵심적인 역할을 한 기술은 **Velite**입니다. 단순한 마크다운 렌더링을 넘어, 옵시디언 마크다운의 메타데이터(Frontmatter)를 기반으로 콘텐츠 스키마를 정의하고 강력한 타입 안정성을 제공해 줍니다.

## 프로젝트 구조
복잡함을 줄이기 위해 '글을 작성하는 환경(content)'과 '화면에 렌더링하는 환경(src)'을 명확하게 분리하여 구성했습니다.

```bash
/
├─ content/          # 옵시디언으로 직접 작성하고 관리하는 마크다운 파일 보관소
│  ├─ post/          # 단일 블로그 포스트
│  ├─ series/        # 이어지는 연재형 콘텐츠
│  ├─ reference/     # 외부 문헌 및 스크랩 참고 자료
│  └─ resource/      # 엑셀, PDF 등 템플릿 첨부파일 중심의 자료실
├─ src/
│  ├─ app/           # Next.js 프레임워크 기반의 페이지 라우팅 로직 (메인, 탐색, 검색 등)
│  ├─ components/    # 템플릿화되어 재사용 가능한 UI 블록들
│  └─ lib/           # 유틸리티 함수 및 Supabase 연동 코드
├─ supabase/         # 백엔드를 대체하는 DB 설정들
└─ velite.config.ts  # 마크다운 파일들을 빌드 시 파싱하고 데이터로 만들지 정의한 스키마 기반 설정 파일
```

이러한 구조 덕분에, 저는 웹사이트 코드를 켤 필요 없이 그저 옵시디언을 열어 `content/` 폴더에 문서만 작성하고 GitHub에 푸시하면 곧바로 블로그에 배포되는 매우 편리한 워크플로우를 갖추게 되었습니다.

## 핵심 구현 기능
단순히 텍스트를 나열하는 것을 넘어, 실무 지식 베이스로서 원활히 작동하도록 다음 기능들을 직접 구현했습니다.

1. 옵시디언 호환 마크다운 렌더링: 가장 공을 들인 부분입니다. `[[문서명]]`과 같은 위키링크(Wiki Link) 형식이 자동으로 내부 HTML 링크로 연결되게끔 커스텀 정규식 변환 로직을 구성했습니다. 또한 `@r4ai/remark-callout` 플러그인을 결합하여 옵시디언에서 즐겨 쓰는 `> [!info]` 형태의 콜아웃 문법이 웹에서도 동일한 디자인으로 예쁘게 출력됩니다.
2. 유형별 맞춤형 페이지 디자인: 단순히 포스트의 카테고리를 나누는 것을 넘어 스키마 단위로 유형을 분리하고, 일반 포스트(Post), 연재물(Series), 외부 문헌(Reference), 자료실(Resource) 등 파일 목적과 성격에 맞춰 레이아웃과 페이지 디자인을 각각 다르게 커스텀 적용하여 정보 전달력을 극대화했습니다.
3. Pagefind를 통한 고성능 전체 검색: 백엔드 데이터베이스 없이 정적 빌드 결과물을 빠르게 인덱싱하는 Pagefind를 결합하여, 사이트 내 어떤 텍스트도 빠르고 가볍게 찾을 수 있는 전역 검색 기능을 마련했습니다.
4. 문서 간 지식 그래프(Knowledge Graph) 구현: `react-force-graph-2d` 라이브러리를 사용해 옵시디언 고유의 경험인 '지식 노드 간의 방향성 연결(태그, 로컬 링크)'을 2D 환경에서 상호작용 가능한 그래프로 구현했습니다.
5. 디자인 제약 탈피와 다크모드 토글 버튼: 주어지는 템플릿 테마에 종속되는 것을 완전히 벗어나, Tailwind CSS와 Framer Motion을 기반으로 감각적인 애니메이션을 직접 적용했습니다. 특히 사이트 전반에 걸쳐 사용자가 원할 때 직관적으로 즉각 전환할 수 있는 다크모드/라이트모드 테마 토글(Toggle) 스위치를 매끄럽게 완성했습니다.
6. 뉴스레터 구독 시스템 구축: 정적 사이트(SSG)의 한계를 넘어 Supabase를 백엔드로 연동함으로써, 방문자가 이메일을 입력해 뉴스레터를 구독할 수 있는 동적 데이터베이스 기반의 리드 캡처(Lead Capture) 기능도 사이트 내에 통합했습니다.
7. 시각적인 My Desk 전용 공간: 평소 실무와 작업에 실제로 즐겨 쓰는 듀얼 모니터 데스크 셋업, 각종 생산성 툴 및 물리적 기기기 등을 한곳에서 시각적인 카드 형태로 소개하고 공유하는 'My Desk' 전용 페이지를 새롭게 제작했습니다.

## 한계점 및 향후 방향성
- 통합 챗봇 에이전트 강화: 기본적으로 RAG(검색 증강 생성) 기능을 연동한 챗봇을 탑재해두었으나, 향후 콘텐츠 라이브러리가 충분히 쌓이면 RAG 기술을 고도화하여 내 블로그의 지식만을 정확하게 찾아 대답해 주는 전용 챗봇으로 발전시킬 계획입니다.
- GraphRAG 및 Ontology 도입: 궁극적으로 GraphRAG와 온톨로지(Ontology) 개념을 도입해 지식 조각들의 관계성을 더 입체적으로 강화하고 싶습니다.
- 양질의 콘텐츠 누적: 지식 베이스의 생명이자 본질은 결국 '글'이므로, 꾸준히 양질의 실무 지식을 기록하고 공유하는 데 주력할 예정입니다.

끝으로, 개발 베이스가 깊지 않으셨던 분들도 AI 코딩 어시스턴트들과 함께라면 충분히 원하시는 모든 기능들이 담긴 멋진 플랫폼을 만들어 보실 수 있을 것이라 생각합니다.
