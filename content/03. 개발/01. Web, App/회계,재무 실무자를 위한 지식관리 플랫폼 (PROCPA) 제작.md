## 1. 제작 동기

### 옵시디언 마크다운을 그대로 웹으로

평소 옵시디언(Obsidian)을 개인 지식관리 도구로 사용하고 있습니다. 업무에 필요한 자료들 뿐만 아니라 블로그에 작성하는 글들도 모두 마크다운으로 작성하고 있는데, 이 콘텐츠를 **별도의 변환 과정 없이** 그대로 웹페이지로 만들 수 있으면 좋겠다는 생각이 출발점이었습니다.

특히 옵시디언의 핵심 기능들 — **그래프 뷰, 백링크, 위키링크, 메타데이터 기반 탐색** — 을 웹에서도 그대로 사용하고 싶었습니다. 옵시디언에서 글을 쓰고 파일만 옮기면 사이트에 바로 반영되는 그런 매끄러운 워크플로우를 원했습니다.

### 퍼스널 브랜딩과 전문성

단순한 노트 공개가 아니라, **회계·재무 전문가로서의 퍼스널 브랜딩**을 담을 수 있는 공간이어야 했습니다. 디자인적으로 심플하면서도 전문적인 인상을 줄 수 있는 사이트를 목표로 삼았습니다.

### 기존 대안의 한계

옵시디언을 활용한 디지털 가든(Digital Garden)을 만드는 방법은 이미 여러 가지가 있습니다.

- **Obsidian Publish** — 옵시디언 공식 퍼블리시 서비스
- **Quartz** — 옵시디언 마크다운을 정적 사이트로 변환하는 오픈소스

하지만 이들은 디자인이 **획일적**이고 커스터마이징에 한계가 있었습니다. 전문가로서의 아이덴티티를 표현하려면 남들과 같은 틀에서 벗어나야 했고, 원하는 기능도 더 필요했기에 직접 만들기로 결정했습니다.

## 2. 개발 도구

### Claude Code + 디자인 스킬

이 프로젝트는 **Claude Code**를 핵심 개발 도구로 활용했습니다. 기획안과 요구사항을 정리한 뒤, 계획(Plan) 모드를 통해 프로젝트의 뼈대를 잡아 나갔습니다.

특히 디자인 작업에는 Claude Code의 **프론트엔드 디자인 스킬**을 적극 활용했습니다. 개인적으로 다크 모드를 선호하여, **Vercel의 디자인 시스템**을 레퍼런스로 삼았습니다. 먼저 디자인 가이드라인 문서를 작성하여 전체적인 톤과 규칙을 정의한 뒤, 이를 기반으로 각 페이지와 컴포넌트를 구현해 나갔습니다.

**디자인 원칙:**

- 순수 블랙(`oklch(0 0 0)`) 다크 모드 + 블루 Primary
- 그라디언트, 글래스모피즘 사용 금지
- **타이포그래피**와 hairline 구분선(`border-border/60`)으로 위계 형성
- **모노스페이스 라벨**로 메타 정보 표시 (날짜, 카테고리, 통계 등)
- 한국어 텍스트에 `word-break: keep-all` 적용

## 3. 기술 스택

| 영역 | 기술 | 선택 이유 |
|---|---|---|
| **프레임워크** | Next.js 16 (App Router) | SSG/SSR 유연성, Turbopack으로 빠른 개발 환경 |
| **런타임** | React 19, TypeScript | 최신 서버 컴포넌트, 타입 안정성 |
| **콘텐츠** | Velite | 마크다운 → typed JSON, Zod 검증, remark/rehype 플러그인 |
| **스타일링** | Tailwind CSS v4 + shadcn/ui | 유틸리티 퍼스트 + 일관된 컴포넌트 시스템 |
| **폰트** | Pretendard (본문) + Geist Mono (라벨) | 한글 최적화 + 기술적 톤의 모노스페이스 |
| **검색** | MiniSearch (커맨드 팔레트) + SSR 검색 | 클라이언트 즉시 검색 + 고급 필터 |
| **그래프** | react-force-graph-2d | 위키링크 기반 지식 그래프 시각화 |
| **코드** | Shiki (rehype-pretty-code) | 다크/라이트 테마 동시 지원 |
| **배포** | Vercel | Next.js 네이티브 지원, 글로벌 CDN |

이 중 핵심은 **Velite**입니다. 단순한 마크다운 렌더링을 넘어, 옵시디언 마크다운의 frontmatter를 Zod 스키마로 검증하고, 위키링크를 파싱해 그래프 데이터를 생성하며, 읽기 시간과 목차를 자동 계산합니다. 빌드 결과물은 TypeScript 타입이 붙은 JSON으로, `import { posts, series, chapters } from '#site/content'` 한 줄로 모든 콘텐츠에 타입 안전하게 접근할 수 있습니다.

## 4. 프로젝트 구조

```
content/                        # 콘텐츠 (옵시디언에서 작성)
├── 00. 인사이트/               # 독립 포스트 (서브카테고리 없이 직접 배치)
├── 01. 회계실무/
│   └── 01. ICFR/              # 서브카테고리 → 시리즈, 포스트
├── 02. AI&생산성/
│   └── 01. AI 활용/           # 서브카테고리 → 시리즈, 포스트
├── 03. 개발/
│   ├── 01. Web, App/
│   └── 02. MCP/
└── downloads/                  # 자료실 (첨부파일 메타)

src/
├── app/                        # Next.js App Router
│   ├── [...slug]/             # 카테고리/포스트/시리즈/챕터 (catch-all)
│   ├── downloads/             # 자료실 페이지
│   ├── explore/               # 탐색 (faceted search)
│   ├── graph/                 # 지식 그래프
│   ├── search/                # 고급 검색
│   └── api/og/               # OG 이미지 자동 생성
├── components/
│   ├── mdx/                   # MDX 컴포넌트 (Callout, FileDownload)
│   ├── vault/                 # INDEX 사이드바 트리
│   └── graph/                 # 그래프 시각화
└── lib/
    ├── topics.ts              # 카테고리 라벨/순서 정의
    ├── vault-tree.ts          # 사이드바 트리 빌드
    └── velite/                # remark 플러그인 (wiki-link, callout, graph)
```

**콘텐츠 파이프라인 흐름:**

```
옵시디언에서 마크다운 작성
    ↓
content/ 폴더에 파일 배치
    ↓
Velite 빌드 (npm run dev 시 자동 실행)
  · frontmatter 검증 (Zod 스키마)
  · MDX 컴파일
  · [[위키링크]] 파싱 → 그래프/백링크 JSON 생성
  · 읽기 시간, TOC, 태그 자동 계산
  · 카테고리 키 자동 추출 (폴더명 → slugify)
    ↓
.velite/ 에 typed JSON 출력
    ↓
Next.js가 JSON import → 페이지 렌더링
```

### 카테고리 자동 매핑

폴더명에서 카테고리 키가 자동으로 추출됩니다. 번호 접두사(`00.`, `01.` 등)는 폴더 정렬용이며 URL에서는 자동 제거됩니다.

| 폴더명 | 카테고리 키 | URL |
|---|---|---|
| `00. 인사이트` | `인사이트` | `/인사이트` |
| `01. 회계실무` | `회계실무` | `/회계실무` |
| `02. AI&생산성` | `ai-생산성` | `/ai-생산성` |
| `03. 개발` | `개발` | `/개발` |

새 카테고리를 추가하려면 `content/` 아래에 폴더를 만들기만 하면 됩니다. 코드 수정 없이 자동으로 인식됩니다. UI 라벨과 정렬 순서만 `src/lib/topics.ts`에 한 줄 추가하면 끝입니다.

## 5. 주요 기능 및 디자인

### INDEX 사이드바

옵시디언의 파일 탐색기를 웹으로 옮겼습니다. 카테고리와 서브카테고리를 폴더 트리 형태로 보여주며, 각 항목의 콘텐츠 수(시리즈는 챕터 수 기준)를 표시합니다. 카테고리를 클릭하면 해당 카테고리 페이지로 이동하여 시리즈 카드와 포스트 목록을 탐색할 수 있습니다.

### 위키링크와 백링크

옵시디언의 `[[위키링크]]` 문법을 그대로 지원합니다. Velite 빌드 시 커스텀 remark 플러그인이 위키링크를 파싱하여 내부 링크로 변환하고, 동시에 백링크 인덱스를 생성합니다. 링크에 마우스를 올리면 대상 문서의 미리보기 툴팁이 표시되며, 포스트 하단에서는 "이 글을 참조한 문서" 목록을 확인할 수 있습니다.

### 지식 그래프

위키링크로 연결된 문서 관계를 시각적으로 표현합니다. 카테고리별 색상 구분과 필터링을 지원하며, 노드를 클릭하면 해당 문서로 이동합니다.

- **Global Graph** (`/graph`) — 전체 문서의 연결 관계
- **Local Graph** — 각 포스트/챕터 페이지에서 인접 노드만 표시

### 콜아웃(Callout)

옵시디언의 콜아웃 문법을 그대로 사용합니다. 커스텀 remark 플러그인이 blockquote 기반의 `> [!note]`, `> [!warning]` 등을 감지하여 스타일이 적용된 컴포넌트로 변환합니다. 13가지 타입과 접기/펼치기(`+`, `-`) 기능을 지원합니다.

### 탐색(Explore)

모든 콘텐츠를 카테고리, 타입(포스트/시리즈/챕터), 태그, 연도 등으로 필터링할 수 있는 faceted search 인터페이스입니다.

### 검색

두 가지 검색을 제공합니다.

- **커맨드 팔레트** (`Ctrl+K`) — MiniSearch 기반 즉시 검색. 제목, 설명, 태그를 대상으로 클라이언트에서 실행
- **고급 검색** (`/search`) — 키워드 + 타입 + 태그 + 날짜 범위를 조합하는 서버 사이드 검색

### 다운로드 카드와 자료실

포스트 본문에서 `/files/`로 향하는 마크다운 링크(`[이름](/files/파일명)`)를 자동으로 다운로드 카드 UI로 변환합니다. 파일 확장자에 따라 아이콘(엑셀, PDF, 이미지 등)이 달라지고 클릭하면 파일이 다운로드됩니다.

별도의 자료실 페이지(`/downloads`)에서는 모든 첨부파일을 카테고리별로 한눈에 볼 수 있습니다. Velite 컬렉션으로 관리되어 제목, 설명, 태그를 포함한 메타 정보와 함께 표시됩니다.

### SEO

- 각 포스트/챕터에 대해 **Open Graph 이미지를 자동 생성** (`/api/og`)
- **JSON-LD** 구조화 데이터 (BlogPosting + BreadcrumbList)
- **sitemap**, **robots.txt**, **RSS 피드** 자동 생성

## 6. 사용 방법

### 포스트 작성

옵시디언에서 마크다운 파일을 작성한 뒤, `content/` 아래 원하는 카테고리에 배치합니다.

**카테고리 바로 아래 (서브카테고리 없이):**

```
content/00. 인사이트/글-제목.md
```

**서브카테고리 아래:**

```
content/01. 회계실무/01. ICFR/글-제목.md
```

파일 앞부분에 frontmatter를 작성합니다:

```yaml
---
title: 글 제목
description: 요약 설명 (300자 이내)
date: 2026-04-13
tags: [태그1, 태그2]
---

본문을 마크다운으로 작성합니다.

[[다른 글 제목]]으로 위키링크를 사용할 수 있습니다.

> [!tip] 콜아웃 예시
> 옵시디언 콜아웃 문법이 그대로 동작합니다.
```

### 시리즈 작성

여러 챕터로 구성된 시리즈(ebook 형태)를 만들려면, 서브카테고리 아래에 시리즈 폴더를 만들고 `index.md`로 메타 정보를 작성합니다.

```
content/02. AI&생산성/01. AI 활용/시리즈 제목/
├── index.md          # 시리즈 메타 (title, description, tags)
├── 1. 소개.md        # 챕터 1 (번호 접두사로 자동 정렬)
├── 2. 설치.md        # 챕터 2
└── 3. 활용법.md      # 챕터 3
```

### 첨부파일 등록

1. `public/files/`에 파일을 저장합니다.
2. `content/downloads/`에 메타 파일을 작성합니다:

```yaml
---
title: DCF 평가 템플릿
description: 클로드 엑셀로 작성한 DCF 모델링 템플릿
file: /files/dcf_test_v2.xlsx
category: ai-생산성
tags: [DCF, 엑셀]
date: 2026-04-13
---
```

3. 포스트 본문에서 링크하면 자동으로 다운로드 카드가 됩니다:

```markdown
[DCF 평가 템플릿](/files/dcf_test_v2.xlsx)
```

### 빌드 및 배포

```bash
npm run dev          # 로컬 개발 서버 (Velite 자동 빌드 포함)
npm run build        # 프로덕션 빌드
```

콘텐츠를 수정하면 Velite가 자동으로 다시 빌드합니다. 저장소에 푸시하면 Vercel이 자동 배포합니다.
